# NetWatch v5.5 内测版

这是 NetWatch 第一个适合公开内测的版本。

NetWatch 是一个 macOS 菜单栏网络体检工具。它会自动判断网络问题出在哪一层：路由器、国内网站、代理软件、代理通道，还是 AI 服务访问。也可以一键查看当前代理 IP 的风险情况。

## 下载安装

下载本页附件里的 `NetWatch-v5.5.dmg`，双击打开后，把「网络体检」拖进「应用程序」。

第一次打开时，请在「应用程序」里右键点「网络体检」→「打开」→ 再点一次「打开」。

## 这版有什么

- 菜单栏常驻小图标，点一下看状态
- 默认 15 秒自动体检，也支持 30 秒 / 1 分钟
- 自动识别 Surge、Shadowrocket、Clash Verge、ClashX、Loon、Quantumult X、V2RayX、sing-box、Hiddify 等代理软件
- 自动判断“代理软件没开”“代理端口没开”“代理通道异常”“宽带/运营商异常”等情况
- 账号安全页：查看当前代理 IP 风险分数、AI 服务可访问状态、出口位置历史
- App 内检查更新和自动替换
- 无需安装 Xcode、brew 或其他额外命令行环境

## v5.5 修复重点

- 修复自动更新先删旧 App 再复制新 App 的风险，改为先复制临时 App、验证后再替换
- 修复安全页切 Tab 后按钮状态丢失导致的并发体检问题
- 修复自动体检和手动体检可能同时写 `风险评估.json` 的问题
- 修复 Shadowrocket 小组件 / Clash 已安装但未运行时被误识别为主程序运行的问题
- 修复代理识别在坏编码进程列表下可能出现 `awk` 报错的问题
- 清理未展示的 timeline 死算，减少后台刷新浪费

## 已知限制

- 目前还没有 Apple 开发者证书和公证，首次打开需要右键打开一次
- 纯 TUN / 增强模式如果没有开放本地 HTTP 或 SOCKS 端口，只能提示“代理软件在跑，但端口没开”
- 如果 App 被系统权限安装到不可写位置，自动更新会安全失败并保留旧版本
